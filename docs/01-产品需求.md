# 桌面待办与历史剪贴板——产品需求文档

> 文档版本：V1.0  
> 产品阶段：v1.0.0正式封板  
> 目标平台：Windows 10 / Windows 11  
> 正式安装名称：小狗效率屋

## v1.0.0封板说明（2026-07-12）

- v1.0.0以阶段5B已经验证的功能作为正式发布基线；
- 已交付待办闭环、文字和图片剪贴板历史、搜索、逐条复制、文字编辑、去重、置顶、取消置顶、单条删除、自定义名称及悬浮窗口；
- 原需求中的1天、2天、4天保存期限、自动清理、批量清空和开机启动，经用户确认延期，不属于v1.0.0验收阻断项；
- 文档后文保留这些需求作为后续版本候选，凡与本节冲突，以本节的正式封板范围为准；
- 本机数据当前未加密，文字和图片不会上传云端，但可能保存密码、验证码、密钥和敏感截图。

## 1. 文档目的

本文档用于明确“桌面待办与历史剪贴板”第一版的产品需求、功能范围、页面结构、交互规则、数据与隐私要求及验收标准，供后续产品设计、界面设计、开发和测试使用。

本文档只描述用户能够看到和使用的功能，不规定开发语言、数据库、界面框架或其他技术实现方案。

> 当前阶段5B体验版已完成待办闭环、文字剪贴板闭环，以及图片保存、重复合并、逐条复制、置顶、取消置顶和单条删除。保存期限、自动清理与批量清空仍未实现，不代表第一版全部验收项已经完成。

## 2. 产品概述

### 2.1 产品定位

“桌面待办与历史剪贴板”是一款运行在 Windows 电脑上的本地个人效率工具，将每日待办和近期剪贴板历史集中在一个可自由摆放、可折叠的桌面悬浮面板中。

软件平时以用户上次选择的位置和大小显示；用户可以拖动、缩放或折叠为小贴纸，在减少桌面占用的同时管理待办或查找近期复制过的文字和图片。

### 2.2 目标用户

- 经常在电脑上复制和粘贴文字、截图或图片的用户；
- 容易丢失此前复制内容，希望快速找回剪贴板历史的用户；
- 希望在桌面上快速记录和查看每日待办的个人用户；
- 不希望注册账号或将个人数据上传到云端的用户。

### 2.3 核心价值

1. **快速找回复制内容：**集中保存近期复制的文字和图片，避免新内容覆盖系统剪贴板后无法找回旧内容。
2. **减少桌面占用：**软件可以自由调整大小并折叠成小贴纸，需要时再展开。
3. **集中处理日常事项：**用户不必打开单独的任务管理软件，即可快速维护每日待办。
4. **本地保存数据：**无账号、无云同步，所有数据只保存在当前电脑中。

### 2.4 第一版目标

第一版应实现以下完整使用闭环：

- 用户打开软件后，软件自动记录运行期间复制的文字和图片；
- 用户能够在剪贴板历史中搜索、逐条重新复制、编辑文字记录、置顶或删除内容；
- 用户能够添加、编辑、完成和删除待办；
- 用户能够折叠或展开桌面侧边面板；
- 普通剪贴板记录能够按照用户设置的期限自动清理；
- 软件退出后停止记录剪贴板。

- macOS、Linux 或移动端版本；

- 多用户或团队协作；
- 剪贴板敏感信息自动识别、过滤或脱敏。

## 4. 整体页面与导航

### 4.1 页面组成

软件包含以下主要界面：

1. **首次启动隐私提示；**
2. **可拖动的折叠贴纸；**
3. **展开后的主面板；**
4. **待办标签页；**
5. **剪贴板标签页；**
6. **待办历史页面或区域；**
7. **设置页面或设置弹层；**
8. **删除、清空等二次确认弹窗。**

### 4.2 主面板导航

展开后的主面板顶部显示两个标签：

- **待办**
- **剪贴板**

用户点击标签后切换相应内容。同一时间只显示一个标签页的内容，以便在有限宽度内提供足够的操作空间。

### 4.3 默认打开页面

- 软件首次运行时，默认显示“待办”标签页；
- 同一次软件运行期间，再次展开面板时，保持用户上一次查看的标签页；
- 软件完全退出并重新打开后，可以再次默认显示“待办”标签页。

## 5. 桌面面板需求

### 5.1 折叠状态

- 软件启动后恢复上次保存的位置和尺寸，并默认以展开状态显示；
- 折叠贴纸始终显示在其他普通应用窗口上方；
- 折叠贴纸应尽量少占用桌面空间，并允许用户继续拖动；
- 折叠贴纸应提供明确的产品图标、拖动区域和展开入口；
- 用户点击贴纸后，主面板在贴纸当前位置展开，并恢复折叠前尺寸。

### 5.2 展开状态

- 展开的主面板允许用户通过顶部区域自由拖动，并通过四边和四角调整大小；
- 主面板始终显示在其他普通应用窗口上方；
- 主面板提供明确的“收起”按钮；
- 点击“收起”按钮后，面板在当前位置折叠为小贴纸；
- 点击面板外部时，面板不自动收起，避免用户操作过程中意外关闭；
- 第一版不提供全局快捷键展开或收起面板。
- 软件只保存展开窗口的位置、尺寸和显示器信息，不保存折叠状态；重启后默认展开。
- 保存位置不可见、显示器被移除或设置损坏时，窗口回退到主屏幕可见区域。
- 用户可通过“重置窗口”恢复默认尺寸和主屏幕右下位置。

### 5.3 关闭和退出

- 软件窗口提供关闭入口；
- 用户点击关闭后，软件应彻底退出，而不是隐藏到系统托盘；
- 软件彻底退出后，不再监听或记录新的剪贴板内容；
- 已经保存在本机的数据不得因正常退出而丢失；
- 下次启动时应恢复此前仍在保留期限内的数据。

### 5.4 开机自动启动

- 软件默认不随 Windows 开机启动；
- 设置中提供“开机自动启动”开关；
- 用户主动开启后，下一次登录 Windows 时自动启动软件；
- 用户关闭该开关后，下一次登录 Windows 时不自动启动；
- 开机启动不改变其他功能和数据。

## 6. 待办功能需求

### 6.1 今日待办列表

“待办”标签页显示当前需要处理的待办事项，包括：

- 今天新添加的待办；
- 之前日期遗留且尚未完成的待办。

每条待办至少显示：

- 待办文字内容；
- 完成勾选入口；
- 编辑入口；
- 删除入口。

### 6.2 添加待办

- 用户可以通过输入框输入待办内容；
- 点击“添加”按钮或按 Enter 键后创建待办；
- 内容为空或只包含空格时，不允许创建；
- 添加成功后，新待办立即出现在今日待办列表中；
- 添加成功后，输入框应清空并可继续输入。

### 6.3 编辑待办

- 用户可以对未完成待办进行编辑；
- 编辑时应显示原有内容；
- 保存后，列表立即显示新内容；
- 编辑结果为空或只包含空格时，不允许保存；
- 用户可以取消编辑，取消后保留原内容。

### 6.4 完成待办

- 用户勾选待办后，该事项被标记为已完成；
- 完成事项从当前未完成列表中移出，并进入完成历史；
- 系统记录实际完成日期和时间；
- 完成记录长期保留，直到用户主动删除；
- 第一版不为完成历史设置自动清理期限。

### 6.5 删除待办

- 用户可以删除未完成待办；
- 用户也可以在完成历史中删除已经完成的待办记录；
- 删除后，该记录从本机数据中移除；
- 第一版可以对删除操作直接执行，但必须提供短暂的删除结果反馈；界面设计也可以采用二次确认，只要交互保持一致。

### 6.6 未完成事项自动延续

- 待办不是到期即删除的临时记录；
- 当日期变化后，仍未完成的事项自动继续出现在新的“今日待办”列表中；
- 自动延续不应创建多条重复记录；
- 原始创建时间应保留，便于后续识别事项创建日期；
- 待办持续延续，直到用户完成或删除。

### 6.7 完成历史

- 待办标签页应提供“完成历史”入口；
- 完成历史按完成日期分组，并以时间从新到旧排列；
- 每条历史记录至少显示待办内容和完成时间；
- 用户可以删除单条历史记录；
- 第一版不要求恢复已完成事项，也不要求搜索待办历史。

### 6.8 待办空状态

当没有未完成待办时，应显示简短提示，例如“今天没有待办”，并保留添加待办入口。

## 7. 剪贴板历史功能需求

### 7.1 自动记录条件

软件在以下条件同时满足时记录剪贴板：

- 软件已经启动且仍在运行；
- 剪贴板中出现第一版支持的内容类型；
- 内容能够被软件正常读取。

软件彻底退出后停止记录。第一版不提供单独的暂停记录按钮。

### 7.2 支持的内容类型

第一版支持：

1. **文字**
   - 普通纯文字；
   - 从富文本复制的内容以可搜索、可再次复制的文字形式保存。

2. **图片**
   - 直接复制的图片；
   - 截图工具放入系统剪贴板的图片。

第一版不记录复制的文件或文件夹。

### 7.3 剪贴板卡片

每条剪贴板记录以卡片形式展示。

文字卡片至少显示：

- 文字内容预览；
- 最近复制时间；
- 置顶或取消置顶入口；
- 删除入口；
- 重新复制入口；
- 文字编辑入口。

图片卡片至少显示：

- 图片缩略图；
- 最近复制时间；
- 置顶或取消置顶入口；
- 删除入口；
- 重新复制入口。

较长文字可以截断预览，但用户应能够查看完整内容。

### 7.4 排列规则

剪贴板列表分为两个区域：

1. **置顶记录区域：**位于普通记录之前；
2. **普通记录区域：**位于置顶记录之后。

两个区域内部均按“最近一次复制时间”从新到旧排列。

### 7.5 重复内容处理

当用户再次复制已经存在的相同内容时：

- 不创建新的重复卡片；
- 更新该记录的最近复制时间；
- 将该记录移动到所属区域的最前方；
- 如果原记录已经置顶，重复复制后仍保持置顶；
- 文字按实际内容判断是否相同；图片按图片内容判断是否相同。

### 7.6 重新复制

- 用户点击剪贴板卡片的主要区域后，该条内容重新写入 Windows 系统剪贴板；
- 操作成功后，应显示明确但不打扰用户的反馈，例如“已复制”；
- 用户随后自行切换到目标软件并按 `Ctrl+V` 粘贴；
- 软件不得自动切换应用、模拟按键或直接向其他软件执行粘贴；
- 重新复制操作会将该记录的最近复制时间更新，并按排列规则移动；
- 软件自身写入剪贴板引发的监听事件不得生成重复记录；
- 每一条历史记录都必须保留独立的重新复制入口，不能只允许重新复制最新一条。

#### 7.6.1 编辑文字记录

- 每一条文字历史记录都可以进入编辑状态；
- 编辑框应显示该条记录的完整文字，而不是截断摘要；
- 保存后，该卡片的预览、搜索内容和后续重新复制内容均更新；
- 空文字或只包含空白字符的编辑结果不允许保存；
- 用户可以取消编辑并恢复原内容；
- 图片记录支持逐条重新复制，但第一版不提供图片编辑器。

### 7.7 置顶和取消置顶

- 用户可以置顶文字或图片记录；
- 置顶后，记录进入置顶区域；
- 置顶记录不受1天、2天、4天保存期限影响；
- 置顶记录一直保留，直到用户取消置顶或主动删除；
- 用户取消置顶后，该记录恢复为普通记录；
- 取消置顶后的到期判断以其最近复制时间为准；如果已经超过当前保存期限，应立即清理或明确提示后清理。

### 7.8 删除单条记录

- 用户可以删除任意一条普通或置顶记录；
- 删除后，记录从列表和本机存储中移除；
- 删除剪贴板历史记录不应清空当前 Windows 系统剪贴板；
- 删除后应提供简短的结果反馈。

### 7.9 搜索

- 剪贴板标签页提供搜索框；
- 搜索仅匹配文字记录的文字内容；
- 搜索不区分置顶和普通记录，但结果仍保持“置顶在前、时间降序”的排列规则；
- 搜索不对图片执行OCR或图片内容识别；
- 输入关键词后，应实时显示或通过明确的搜索操作显示匹配结果；
- 清空搜索关键词后，恢复完整列表；
- 无匹配结果时，显示“未找到相关内容”等空状态提示。

### 7.10 图片异常

- 如果图片记录存在但缩略图暂时无法显示，卡片应显示“图片无法加载”等占位状态；
- 单条图片加载失败不得导致整个剪贴板列表无法使用；
- 用户仍可以删除无法加载的记录；
- 如果原始图片数据仍可读取，应允许用户尝试重新复制；无法读取时应提示操作失败。

### 7.11 剪贴板空状态

当没有可显示的剪贴板记录时，应显示简短提示，例如“还没有剪贴板记录”。

## 8. 保存期限和自动清理

### 8.1 保存期限设置

设置中提供以下三个固定选项：

- 1天；
- 2天；
- 4天。

默认值为 **2天**。

第一版不提供任意天数输入，也不按记录数量限制历史。

### 8.2 自动清理规则

- 保存期限只对未置顶的剪贴板记录生效；
- 到期时间以记录的“最近复制时间”为起点计算；
- 达到保存期限后，普通记录自动从列表和本机存储中删除；
- 软件启动时应检查并清理已经到期的普通记录；
- 软件运行期间也应定期或在适当时机清理到期记录；
- 自动清理不得删除任何仍处于置顶状态的记录。

### 8.3 修改期限

- 用户修改保存期限后，新期限对现有未置顶记录和未来记录同时生效；
- 如果用户将期限从较长时间改为较短时间，已经超过新期限的普通记录应及时清理；
- 修改保存期限不得影响置顶记录或待办数据。

### 8.4 一键清空历史

- 剪贴板页面或设置中提供“一键清空剪贴板历史”入口；
- 点击后必须显示二次确认，明确说明将删除所有未置顶记录；
- 用户确认后，删除全部未置顶文字和图片记录；
- 置顶记录必须保留；
- 用户取消时，不删除任何内容；
- 清空操作不应清空当前 Windows 系统剪贴板；
- 完成后显示清空成功反馈。

## 10. 数据与隐私需求

### 10.1 本地存储

- 所有待办、剪贴板记录和设置只保存在当前电脑本机；
- 第一版不要求用户创建账号或登录；
- 第一版不得将剪贴板、待办或使用数据上传到云端；
- 第一版不提供跨电脑同步；
- 卸载软件时如何处理本地数据，可在后续安装方案中确定，但不得在用户不知情的情况下上传数据。

### 10.2 敏感内容

软件不主动识别或过滤密码、验证码、密钥、身份证号码等敏感内容。只要这些内容进入系统剪贴板且属于支持类型，就可能被记录。

因此，首次启动时必须向用户显示隐私提示，至少说明：

- 软件会记录运行期间复制的文字和图片；
- 记录中可能包含密码、验证码等敏感内容；
- 数据仅保存在当前电脑本机；
- 用户可删除单条记录或清空未置顶记录；
- 点击关闭并彻底退出软件后，将停止记录。

用户确认隐私提示后才能进入主界面。确认状态可保存在本机，后续启动不必重复显示；产品可以在设置中提供再次查看隐私说明的入口。

### 10.3 数据隔离

第一版按当前 Windows 用户使用本机数据，不考虑多个产品账号或团队之间的数据共享。

## 11. 核心用户流程

### 11.1 首次使用

1. 用户启动软件；
2. 软件显示首次启动隐私提示；
3. 用户确认后进入主面板；
4. 面板显示“待办”和“剪贴板”标签；
5. 用户收起面板后，软件在当前位置以小贴纸形式存在；
6. 软件在运行期间开始记录支持的剪贴板内容。

### 11.2 添加并完成待办

1. 用户点击折叠贴纸展开面板；
2. 进入“待办”标签页；
3. 输入待办内容并添加；
4. 待办出现在当前列表；
5. 用户完成后勾选该事项；
6. 事项从未完成列表移出并进入完成历史。

### 11.3 找回并重新使用文字

1. 用户展开面板并进入“剪贴板”标签页；
2. 在搜索框输入关键词；
3. 列表显示匹配的文字记录；
4. 用户点击目标卡片；
5. 软件提示“已复制”；
6. 用户返回目标应用，按 `Ctrl+V` 粘贴。

### 11.4 找回图片

1. 用户展开剪贴板标签页；
2. 按时间浏览图片缩略图；
3. 点击目标图片卡片；
4. 图片被重新写入系统剪贴板；
5. 用户在目标应用中按 `Ctrl+V` 粘贴。

### 11.5 保护重要记录

1. 用户找到需要长期保留的文字或图片；
2. 点击“置顶”；
3. 记录移动到置顶区域；
4. 即使超过保存期限，该记录仍然保留；
5. 用户取消置顶或删除后，记录恢复普通清理规则或被移除。

### 11.6 彻底退出

1. 用户点击软件关闭入口；
2. 软件保存当前数据和设置；
3. 软件进程彻底退出；
4. 后续复制内容不再被记录；
5. 用户再次启动软件后，恢复未过期记录、置顶记录、待办和设置。

## 12. 异常与边界规则

| 场景 | 预期行为 |
| --- | --- |
| 复制空文字 | 不创建记录 |
| 复制只包含空格的文字 | 不创建记录 |
| 连续复制相同内容 | 合并为一条，更新时间并移到所属区域最前 |
| 软件自身重新复制历史内容 | 不创建重复记录 |
| 复制不支持的文件或文件夹 | 不记录，不影响软件正常使用 |
| 剪贴板暂时被其他程序占用 | 本次读取失败不得导致软件崩溃；后续复制仍可继续记录 |
| 图片缩略图加载失败 | 显示占位状态，其他记录继续正常展示 |
| 普通记录到期 | 自动删除 |
| 置顶记录到期 | 不删除 |
| 已过期记录取消置顶 | 按当前期限立即清理或给予明确提示后清理 |
| 清空历史 | 二次确认，只删除未置顶记录 |
| 删除历史记录 | 不改变当前系统剪贴板内容 |
| 日期跨天 | 未完成待办继续显示，不创建重复待办 |
| 软件退出 | 停止记录，已保存数据保留 |
| 开机启动关闭 | Windows 登录后不自动启动 |
| 搜索无结果 | 显示明确的空状态提示 |
| 本地数据为空 | 待办和剪贴板页面分别显示空状态 |

## 13. 验收标准

### 13.1 桌面面板

- [ ] 软件可在 Windows 10 和 Windows 11 上启动；
- [ ] 折叠时在当前位置显示为可拖动小贴纸；
- [ ] 折叠贴纸和展开面板始终位于普通窗口上方；
- [ ] 点击折叠贴纸能够在当前位置展开主面板并恢复原尺寸；
- [ ] 点击收起按钮能够折叠为小贴纸；
- [ ] 点击面板外部不会自动收起；
- [ ] 用户可以拖动展开面板和折叠贴纸；
- [ ] 用户可以通过四边和四角调整展开面板大小，且不能小于最低可用尺寸；
- [ ] 关闭重启后恢复上次展开位置和尺寸，但以展开状态启动；
- [ ] 显示器变化或设置损坏时，窗口仍回到可见工作区；
- [ ] “重置窗口”能够恢复默认尺寸和主屏幕右下位置；
- [ ] 主面板可在“待办”和“剪贴板”标签之间切换；
- [ ] 点击关闭后软件彻底退出，且不再记录剪贴板。

### 13.2 待办

- [ ] 用户可以添加非空待办；
- [ ] 用户可以编辑未完成待办；
- [ ] 用户可以完成待办；
- [ ] 用户可以删除未完成待办和完成历史；
- [ ] 未完成待办跨天后仍然显示，且不会产生重复记录；
- [ ] 完成事项进入按完成时间降序展示的历史；
- [ ] 软件重启后待办和完成历史仍然存在。

### 13.3 剪贴板记录

- [ ] 软件运行时能够记录复制的文字；
- [ ] 软件运行时能够记录复制或截图产生的图片；
- [ ] 软件不记录文件和文件夹；
- [ ] 普通记录按最近复制时间降序展示；
- [ ] 相同内容重复复制时只保留一条，并更新时间；
- [ ] 每一张文字或图片卡片都能够将自身对应内容重新写入系统剪贴板，而不是只操作最新一条；
- [ ] 每一张文字卡片都能够编辑完整内容，保存后搜索和重新复制使用新内容；
- [ ] 图片卡片可重新复制，但不显示文字编辑入口；
- [ ] 重新复制不会产生重复卡片；
- [ ] 软件退出后，新复制内容不会被记录。

### 13.4 搜索

- [ ] 用户能够通过关键词搜索文字记录；
- [ ] 搜索结果只包含文字内容匹配的记录；
- [ ] 图片内容不会被OCR识别或搜索；
- [ ] 清空关键词后恢复完整列表；
- [ ] 无搜索结果时显示明确提示。

### 13.5 置顶和删除

- [x] 用户可以置顶或取消置顶文字和图片记录；
- [x] 置顶记录显示在普通记录之前；
- [ ] 置顶记录不会因保存期限到期而删除；
- [x] 用户可以删除普通记录和置顶记录；
- [x] 删除历史记录不会清空当前系统剪贴板。

### 13.6 保存期限和清空

- [ ] 保存期限可选择1天、2天或4天；
- [ ] 初始默认保存期限为2天；
- [ ] 到期的未置顶记录会自动删除；
- [ ] 缩短保存期限后，超过新期限的普通记录会及时清理；
- [ ] 一键清空前显示二次确认；
- [ ] 确认清空后只删除未置顶记录；
- [ ] 取消清空后不删除任何记录。

### 13.7 设置、启动和隐私

- [ ] 软件默认不开机自动启动；
- [ ] 用户可以在设置中开启或关闭开机自动启动；
- [ ] 设置在软件重启后仍然保留；
- [ ] 首次启动时显示剪贴板及敏感信息隐私提示；
- [ ] 用户确认隐私提示后才进入主界面；
- [ ] 待办、剪贴板和设置数据只保存在本机；
- [ ] 软件不要求注册或登录，不提供云同步。

## 14. 非功能要求

### 14.1 易用性

- 界面文字使用简体中文；
- 主要操作应使用清晰的文字、图标或提示表达；
- 添加待办、重新复制、置顶、删除和收起等高频操作应容易找到；
- 空状态、失败状态和操作结果均应提供用户可理解的反馈。

### 14.2 稳定性

- 单条剪贴板内容读取或图片加载失败不得导致软件整体崩溃；
- 软件重启后应能够恢复仍需保留的数据；
- 自动清理不得误删置顶内容或待办数据；
- 长时间运行时应保持正常记录和面板操作能力。

### 14.3 性能体验

- 折叠和展开操作应流畅；
- 待办和剪贴板页面应在用户操作后及时更新；
- 后台记录剪贴板时不应明显影响电脑的正常使用；
- 图片应使用适合面板展示的缩略图，避免列表浏览明显卡顿。

### 14.4 兼容性

- 第一版以 Windows 10 和 Windows 11 为目标平台；
- 应兼容常见应用产生的纯文字和图片剪贴板内容；
- 对无法读取或不支持的剪贴板格式应安全忽略，而不是中断软件运行。

## 15. 已确认的产品规则

1. 产品暂用“桌面待办与历史剪贴板”作为工作名称；
2. 第一版仅支持 Windows 10/11；
3. 所有数据只保存在本机，不需要账号和云同步；
4. 密码、验证码等敏感内容也可能被记录，不做自动过滤；
5. 软件运行时自动记录，彻底退出后停止记录；
6. 不提供单独暂停记录按钮；
7. 面板可自由拖动和缩放，始终置顶；位置与尺寸保存在本机，使用按钮主动收起为当前位置的小贴纸；
8. 待办和剪贴板使用两个标签页；
9. 待办仅实现基础管理，不包含提醒、优先级、分类和备注；
10. 未完成待办自动延续，完成记录长期保留；
11. 剪贴板只支持文字和图片，不支持文件；
12. 搜索只匹配文字，不对图片进行OCR；
13. 重复内容合并并更新最近复制时间；
14. 保存期限固定为1天、2天、4天，默认2天；
15. 置顶内容不自动过期；
16. 一键清空只删除未置顶记录，并且必须二次确认；
17. 每条历史卡片均可独立重新复制；文字卡片可编辑，图片卡片第一版不提供图片编辑器；用户仍自行按 `Ctrl+V` 粘贴；
18. 默认不开机启动，但允许用户在设置中开启；
19. 点击关闭后直接彻底退出，不缩小到系统托盘；
20. 第一版不提供全局快捷键。
