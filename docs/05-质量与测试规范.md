# 质量与测试规范

## 完成定义

一个开发事项只有同时满足以下条件才算完成：

- 行为符合产品需求和当前阶段目标；
- 通过构建、静态检查和相关自动化测试；
- 新增或修改的业务规则有对应测试；
- 已手工验证关键流程和错误状态；
- 未引入第一版范围外功能；
- 相关正式文档和当天开发日志已更新；
- 没有隐藏阻断或高风险缺陷。

## 测试分层

### 单元测试

覆盖待办校验和跨天延续、剪贴板去重和排序、置顶、保存期限、自动清理、搜索匹配等独立规则。

### 集成测试

覆盖数据库读写与迁移、图片文件生命周期、设置持久化、Windows剪贴板适配和开机启动配置。

### 界面与流程测试

覆盖面板展开收起、待办闭环、文字和图片记录、重新复制、搜索、置顶、删除、清空、隐私提示和各类空状态/错误状态。

## 阶段门槛

- 上一阶段测试未通过，不进入下一阶段；
- 缺陷修复应补防复发测试，无法自动化时记录手工步骤；
- 每次提交前运行受影响测试；
- 阶段完成前运行完整测试集；
- 发布候选版执行产品需求中的全部验收项。

## 必测场景

- 快速连续复制相同及不同文字；
- 软件自身重新复制的回流事件；
- 剪贴板被其他应用短暂占用；
- 空文字、仅空格文字和大段文字；
- 截图、透明图、大图和损坏图片；
- 日期跨天、休眠唤醒和应用重启；
- 保存期限由4天缩短到1天；
- 已过期记录取消置顶；
- 清空操作保留所有置顶记录；
- 本地数据异常或磁盘空间不足；
- 常见Windows缩放和分辨率。

## 缺陷等级

- **阻断：**无法启动、数据大面积丢失、误删置顶、无法退出、安全边界突破；
- **高：**核心功能不可用、频繁崩溃、监听死循环、明显漏记或重复；
- **中：**有替代路径但结果或体验不符合需求；
- **低：**轻微视觉或文案问题。

阻断和高等级缺陷未解决时不得发布。

## 测试记录

每日开发日志应记录测试命令或手工步骤、通过和失败结果、未执行项目及原因、新风险和下一步。

## v1.0.0发布门槛（2026-07-12）

- Release构建必须0错误、0警告，现有100项自动验证必须全部通过；
- 必须完成正式便携程序UI烟雾测试，且测试后恢复用户原有应用数据和系统剪贴板；
- 必须验证安装、启动、覆盖安装、单一卸载项、开始菜单、默认不创建桌面快捷方式；
- 必须验证默认卸载保留数据，显式选择删除时只清除固定受管目录；
- 安装包和便携程序必须生成SHA-256，测试报告必须如实列出未覆盖的Windows版本和签名限制。
