# 项目文档索引

本目录保存“桌面待办与历史剪贴板”项目的正式标准。后续开发、设计、测试和评审均以这里的文档为准。

| 文件 | 用途 |
| --- | --- |
| `01-产品需求.md` | 第一版产品范围、功能规则和验收标准 |
| `02-开发路线图.md` | 分阶段开发步骤、阶段出口和当前优先级 |
| `03-技术规范.md` | 架构、数据、错误处理、性能和工程约束 |
| `04-界面与交互设计规范.md` | 布局、状态、交互和可用性标准 |
| `05-质量与测试规范.md` | 测试分层、完成定义和回归要求 |
| `06-安全与隐私规范.md` | 本地数据、敏感剪贴板和安全边界 |
| `07-开发执行规范.md` | 日常工作流程、变更规模和日志要求 |
| `08-技术决策-001-WPF原型.md` | 阶段0技术候选、选择、验证结果、限制和回退方案 |

## 文档优先级

发生冲突时按以下顺序处理：

1. 用户最新明确确认的需求；
2. `01-产品需求.md`；
3. `06-安全与隐私规范.md`；
4. 技术、设计、测试和执行规范；
5. 开发日志中的临时记录。

影响产品范围、数据规则或用户可见行为的变更，必须先更新相应文档，再开始实现。

- `09-技术决策-002-SQLite本地数据层.md`：阶段1 SQLite数据路径、版本、待办仓储与失败保护决策。
- `10-正式版发布与安装规范.md`：v1.0.0版本身份、安装升级、卸载和交付规则。
- `11-v1.0.0测试报告.md`：正式版构建、自动验证、UI和安装生命周期测试结果。
