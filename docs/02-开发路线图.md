# 开发路线图

## 推进原则

- 每次只推进一个可以独立验证的小目标。
- 每个阶段必须满足“阶段出口”后才能进入下一阶段。
- 优先验证高风险能力：Windows剪贴板监听、图片持久化、自由悬浮置顶面板、自动清理。
- 不开发第一版范围外功能。
- 每次开发结束必须更新当天开发日志。

## 阶段0：工程决策与可行性验证（已结束）

**目标：**不开发完整产品，先确认关键技术稳定可行。

已完成：

1. 选定.NET 10、WPF、Windows x64自包含发布；
2. 验证文字和图片剪贴板监听、文件剪贴板忽略与自身回流抑制；
3. 验证自由拖动、缩放、位置记忆、始终置顶、折叠和展开；
4. 完成参考图小狗资源与浅绿、暖灰、米白视觉原型；
5. 完成内存版待办、剪贴板搜索和逐条操作技术验证。

结束说明（2026-07-12）：用户确认阶段0结束并授权进入下一阶段。

## 阶段1：工程骨架与数据层（已完成）

**目标：**建立可运行、可测试、可安全升级的本机数据基础。

已完成：

- 建立SQLite数据库、版本管理、事务迁移和幂等初始化；
- 待办数据本机持久化；
- 数据库损坏、占用或不可写时不覆盖原文件，应用可回退为本次运行有效模式；
- 建立可重复的Release构建、自动验证和Windows x64自包含发布脚本。

**阶段出口：**空壳应用稳定启动退出；数据可持久化并安全升级；核心数据测试通过。

## 阶段2：基础桌面面板与逐条操作（已完成）

已完成：

- 自由悬浮面板、位置尺寸记忆、贴纸折叠、点击展开和按钮收起；
- 始终置顶、边界约束、标签导航、直接关闭退出；
- 每条文字和图片记录均可独立重新复制；
- 每条文字记录可行内编辑；
- 搜索匹配完整文字，图片不执行OCR。

**阶段出口：**窗口无丢失、遮挡关键操作或无法关闭问题；历史逐条复制和文字编辑通过烟雾测试。

## 阶段3：待办完整闭环与个性化名称（已完成）

已完成：

- 添加、编辑、完成、恢复未完成和删除待办；
- 今日待办与完成历史分区；
- 完成时间本机保存，完成历史按日期分组和时间降序；
- 未完成事项跨天和重启继续显示，不复制生成新记录；
- 工具名称可修改、恢复默认并在本机保存；
- 修复勾选时列表提前过滤导致完成状态不可靠的问题。

**阶段出口：**待办闭环通过自动验证和真实鼠标UI烟雾测试，跨天和重启不丢失、不重复。

## 阶段4：文字剪贴板闭环（已完成）

### 阶段4A：文字历史持久化（已完成）

- SQLite升级到版本3，新增文字剪贴板记录表；
- 新文字保存到本机，启动时恢复并按最近复制时间降序显示；
- 相同文字按SHA-256指纹合并，只更新时间并移动到顶部；
- 任意历史文字重新复制成功后更新时间，软件自身回流不新增记录；
- 文字编辑同步持久化；编辑成已有内容时自动合并；
- 搜索继续匹配完整文字并忽略英文大小写；
- 首次启用前显示敏感信息隐私提示，未确认时不监听新的剪贴板内容；
- 图片继续只保存在本次运行内存中，最多30条。

### 阶段4B：文字记录管理（已完成）

- 文字记录支持置顶和取消置顶，置顶记录始终排在未置顶记录之前；
- 同组内继续按最近复制时间降序；
- 重复复制保持原置顶状态；编辑合并时任一记录已置顶，合并结果继续置顶；
- 文字记录支持单条永久删除；数据库失败时保留原卡片；
- 图片不提前增加置顶或删除，继续留待阶段5。

**阶段出口：**文字记录可保存、搜索、合并、重新复制、编辑、置顶和删除，无明显漏记、重复或死循环。

## 阶段5：图片剪贴板闭环（v1.0.0基线已完成）

### 阶段5A：图片保存与复用（已完成）

- 监听截图和复制图片；
- SQLite升级到版本4，保存图片元数据；
- 在应用数据目录安全保存PNG原图并生成最长边不超过320像素的PNG缩略图；
- 实现图片内容去重、重启恢复、缩略图展示和逐条重新复制原图；
- 单张图片写入失败时清理本次新建文件并回退为本次运行有效模式；
- 统一功能卡片与输入框圆角。

### 阶段5B：图片管理（已完成）

- 实现图片置顶、取消置顶和单条删除；
- 图片置顶状态重启后恢复，重复复制不丢失置顶状态；
- 删除采用文件暂存、数据库事务和启动恢复机制，保证原图、缩略图与数据库记录一致；
- 删除时保证数据库记录、原图和缩略图生命周期一致；

### 阶段5C：图片异常与性能收尾（延期到后续版本）

- 补充损坏、缺失、超大图片处理验证；
- 验证较多图片记录下的内存、磁盘占用和列表滚动体验；
- 检查孤立文件与错误状态提示，不提前实现自动到期。

**阶段出口：**常见图片稳定记录和复用，单条失败不影响应用，不执行OCR。

## 阶段6：清理、设置与隐私（经用户确认延期）

- 实现1天、2天、4天保存期限，默认2天；
- 实现启动检查和运行期自动清理；
- 确保置顶记录永不过期；
- 实现清空未置顶记录及二次确认；
- 实现开机启动开关；
- 在已实现首次隐私提示基础上补充再次查看入口和完整设置闭环。

**阶段出口：**自动清理不误删置顶或待办，设置重启后保持，隐私说明符合需求。

## 阶段7：稳定性、打包与发布候选版（v1.0.0已完成）

- 执行完整功能、回归、长时间运行和异常恢复测试；
- 验证高DPI、休眠唤醒、系统重启、安装升级与卸载；
- 修复阻断和高优先级缺陷；
- 生成可重复构建的安装包、版本说明、已知问题和回退方案。

**阶段出口：**第一版验收项全部通过，无阻断或高风险未解决缺陷。

## 当前进度与封板结论

- 2026-07-12，用户确认以阶段5B功能集封板并发布“小狗效率屋 v1.0.0”；
- SQLite保持v4，现有待办、剪贴板和设置路径不迁移、不清空；
- 已完成正式版本元数据、应用图标、当前用户安装包、开始菜单快捷方式、可选桌面快捷方式和卸载数据选择；
- 阶段6的保存期限、自动清理、批量清空和开机启动明确延期，不得标记为已实现；
- v1.0.0封板后不再增加业务功能，仅接受阻断启动、数据损坏、误删或安全边界问题修复；
- 后续开发必须从新版本路线图开始，不得直接修改v1.0.0冻结范围。
