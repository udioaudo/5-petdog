# 技术决策001：阶段0采用.NET 10 WPF原型

> 状态：阶段0暂定采用，等待本地存储验证后最终确认  
> 日期：2026-07-12

## 背景

产品只支持Windows 10/11，需要可靠访问系统剪贴板、控制可自由移动和缩放的置顶窗口，并最终发布不要求用户另装运行时的桌面程序。当前电脑原有Node.js，但没有系统级.NET或Rust开发环境。

## 候选方案

| 方案 | 优点 | 主要风险 |
| --- | --- | --- |
| .NET 10 + WPF | Windows原生窗口和消息能力直接；可发布自包含程序；工程分层和测试成熟 | 仅支持Windows；开发机需要准备.NET SDK |
| Electron | 当前电脑已有Node.js；前端界面开发方便 | 官方剪贴板接口缺少原生变化事件，需要轮询或额外原生模块；运行体积和空闲资源更高 |
| Tauri | 安装包和运行资源通常较小 | 当前电脑没有Rust；仍需开发Windows原生剪贴板桥接，第一阶段工具链成本较高 |

## 决策

阶段0原型采用 **.NET SDK 10.0.301 + WPF**：

- SDK安装在项目 `.tools/dotnet`，不修改系统级环境；
- 原型目标为 `net10.0-windows`；
- 体验包使用 `win-x64` 自包含单文件发布；
- Windows剪贴板通过 `AddClipboardFormatListener` 消息通知监听，不使用高频轮询；
- 当前原型不建立数据库、不写入图片文件，只在内存保留最多30条剪贴板事件；
- 第三版增加的待办、编辑状态和搜索词也只保存在内存中；
- 用户参考图以WPF Resource嵌入程序集，自包含程序不依赖 `E:\7.12.jpg` 外部路径。

## 已验证结果

- Release构建0警告、0错误；
- 36项自动验证通过，新增覆盖文字搜索规则、图片搜索排除、待办输入规范化和状态通知；
- 实际窗口可启动、自由拖动和缩放、始终置顶并保持响应；
- 实际复制文字和图片可捕获，文件剪贴板安全忽略；
- 自身重新复制产生的多个消息可在2秒窗口内抑制，不新增记录；
- 窗口可折叠为56×96 DIP小狗贴纸，重新展开恢复原尺寸；
- 展开窗口位置和尺寸只保存在 `%LocalAppData%\HMinus\DesktopSpike\window-settings.json`；
- 第三版已通过待办添加、完成、编辑、删除和剪贴板搜索的窗口自动化烟雾测试；
- 小狗图片资源随应用加载，第三版视觉改为浅绿、暖灰和米白配色。

## 已知限制

- 混合DPI多显示器和真实显示器插拔仍需更多人工验证；
- 100%、150%、200%缩放仍需用户或后续人工验证；
- 图片指纹会编码PNG并计算哈希，超大图片性能尚未压测；
- 快速连续变化时当前实现串行读取“当时最新剪贴板”，极端情况下可能跳过中间瞬时内容；
- 2秒回流抑制窗口内，手工复制完全相同内容会被视为同一次回流或重复；
- 原型无业务持久化，退出后待办、搜索状态和剪贴板事件消失；
- 搜索只覆盖文字摘要，不执行图片OCR。

## 回退与后续

如果WPF窗口行为、资源打包或后续存储验证出现不可接受问题，阶段0可重新比较Electron或Tauri，不把原型直接升级为正式产品。

下一小步仍只验证本地SQLite数据和图片文件生命周期；验证通过前不把第三版内存交互视为阶段3或阶段4完成。
